## 残課題（優先度順）

### #1 【P0】ステップインジケータがまだない

| | |
|--|--|
| **問題** | モード選択は分かりやすくなったが、「環境確認 → 同盟/ルーム → 役割」の **全体進捗** は依然なし。 |
| **影響** | 1端末ユーザーは時計合わせステップを挟み、どこまで進んだか見失いやすい。 |
| **提案** | 画面上部に `①モード ②環境 ③参加 ④役割` の固定バー（現在をハイライト）。 |
| **工数** | 小 |

---

### #2 【P0】同盟練習の「参加成功」が分かりにくい

| | |
|--|--|
| **問題** | `mode_ok` で `setDrillStatus("")` のあと **無言で** `selectAlliance(0)` → 役割画面（L2590–2606）。成功バナーなし。 |
| **影響** | 「作成して入る」押下後、一瞬何も変わらないように感じる（通信待ち）。 |
| **提案** | 成功時: `drillStatusMsg` に「✓ ルームに参加しました。役割を選んでください」→ 0.5秒後に step3。または step3 遷移時にトースト。 |
| **工数** | 小〜中 |

---

### #3 【P0】オンボーディングの未入力が `alert()` のまま

| | |
|--|--|
| **問題** | 同盟名・参加コード・ルーム・役割・名前で `alert()`（L1319–1320, 1336–1337, 2469–2480）。 |
| **影響** | スマホでダイアログが重く、入力欄の直下にエラーが出ない。 |
| **提案** | `drillStatusMsg` / 入力欄直下の `.field-error` に統一（訓練の通信エラー表示は良い手本）。 |
| **工数** | 中 |

---

### #4 【P1】参謀は依然「2段階」で止まりやすい

| | |
|--|--|
| **問題** | 参謀選択後、`staffPlayerRoleArea` の班選択まで `inputsArea`・登録ボタンが出ない（変更なし）。 |
| **提案** | 「次: 担当する班を選んでください」の1行 + 未選択時は「登録して開始」を薄く表示。 |
| **工数** | 小 |

---

### #5 【P1】SVS の同盟はワンタップ即確定（誤タップ）

| | |
|--|--|
| **問題** | `selectAlliance(idx)` は確認なしで step3 へ（変更なし）。 |
| **提案** | タップでハイライト → 「この同盟で進む」確定ボタン（任意で次回スキップ）。 |
| **工数** | 小 |

---

### #6 【P1】step1 が縦に長い（スマホ）

| | |
|--|--|
| **改善点** | モード選択が **最上部** に来たのは良い（スクロール前に選べる）。 |
| **残り** | その下に Chrome 警告・URL コピー・端末質問が続き、**1画面に収まりきらない** 端末がある。 |
| **提案** | Chrome/URL を折りたたみ「初めての方だけ」；端末質問をモード直後に短縮。 |
| **工数** | 中 |

---

### #7 【P2】カード順序と利用頻度のミスマッチ

| | |
|--|--|
| **問題** | ヒントは「普段は同盟の練習」だが、**上に SVS カード**が来る。 |
| **影響** | 毎日練習するユーザーが毎回上から読む／誤って SVS を選ぶ可能性。 |
| **提案** | ① カード順を「同盟の練習」を上にする、または ② 前回モードを **視覚的に先頭** に並べ替え。 |
| **工数** | 極小〜小 |

---

### #8 【P2】「戻る」ラベルがモード非連動

| | |
|--|--|
| **問題** | step3 の戻るは常に「◀ 同盟の選択に戻る」。同盟練習時はルーム画面のはず。 |
| **提案** | `appMode === 'drill'` のとき「◀ ルーム参加に戻る」。 |
| **工数** | 極小 |

---

### #9 【P2】参加コードの説明不足

| | |
|--|--|
| **問題** | ルーム選択 + 参加コードの両方必須の理由が placeholder のみ。 |
| **提案** | 作成成功時にコードを表示＋コピー；参加側に「参謀から共有されたコードを入力」。 |
| **工数** | 小 |

---

### #10 【P2】訓練タブボタンがやや小さい

| | |
|--|--|
| **問題** | 新規作成/ルーム参加は `padding:8px`（モードカードより小さい）。 |
| **提案** | 最小高さ 44〜48px、フォント 15px。 |
| **工数** | 極小（CSS） |
